# Утилита journalctl

Утилита `journalctl` используется для чтения и фильтрации бинарных логов, собранных демоном `systemd-journald`.

Основные примеры использования:
- `journalctl` (без аргументов): вывод всего журнала, начиная со старых записей.
- `journalctl -u nginx.service`: вывод логов только от конкретной службы.
- `journalctl -f`: режим "follow", вывод новых сообщений в реальном времени (подобно `tail -f`).
- `journalctl -b`: вывод логов только за текущую загрузку системы (текущий boot).
- `journalctl -p err`: вывод только сообщений с приоритетом Error (ошибки) и выше.
