# Ротация логов logrotate

Текстовые лог-файлы в каталоге `/var/log` постоянно растут и со временем могут переполнить диск. Для решения этой проблемы используется утилита `logrotate`.

Она запускается по расписанию (обычно ежедневно через `cron`) и управляет архивированием старых логов.

Процесс ротации:
1. Текущий лог (например, `syslog`) переименовывается в `syslog.1`.
2. Демону логов отправляется сигнал создать новый пустой файл `syslog`.
3. Старые файлы сжимаются утилитой `gzip` (`syslog.2.gz`).
4. Самые старые файлы (в зависимости от настроек, например, старше 7 дней) удаляются навсегда.
