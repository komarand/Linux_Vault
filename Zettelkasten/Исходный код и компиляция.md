# Исходный код и компиляция

Если нужной программы нет в официальных репозиториях, её часто собирают (компилируют) из исходного кода вручную.

Классический процесс сборки программы на языке C/C++ включает три шага:
1. `./configure`: скрипт, проверяющий систему на наличие необходимых библиотек и компиляторов. Он генерирует файл `Makefile`.
2. `make`: утилита, которая читает `Makefile` и запускает компилятор (например, `gcc`) для создания бинарных исполняемых файлов.
3. `sudo make install`: копирует скомпилированные файлы в системные директории (обычно в `/usr/local/bin`).

Этот метод сложнее пакетного менеджера, так как зависимости приходится разрешать вручную.
