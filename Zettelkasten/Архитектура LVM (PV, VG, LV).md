# Архитектура LVM (PV, VG, LV)

Архитектура LVM состоит из трех основных уровней абстракции:

1. **Физические тома (Physical Volumes, PV)**: базовые блочные устройства (жесткие диски или разделы `sdX`), которые инициализируются утилитой `pvcreate` для использования в LVM.
2. **Группы томов (Volume Groups, VG)**: один или несколько `PV` объединяются в единый пул дискового пространства (VG). Это "резервуар", из которого выделяется емкость.
3. **Логические тома (Logical Volumes, LV)**: виртуальные разделы, создаваемые внутри группы томов (`VG`). Именно на `LV` форматируются файловые системы и монтируются в систему. Их размер ограничен только общим размером `VG`.
