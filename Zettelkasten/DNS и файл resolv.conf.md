# DNS и файл resolv.conf

Файл `/etc/resolv.conf` традиционно используется в Linux для указания адресов DNS-серверов, которые будут преобразовывать доменные имена (например, google.com) в IP-адреса.

Простейший вид файла содержит строки `nameserver 8.8.8.8`.
В современных системах этот файл часто является символической ссылкой, управляемой демонами вроде `systemd-resolved` или NetworkManager, которые автоматически перезаписывают его на основе данных, полученных по DHCP. Поэтому ручное редактирование этого файла больше не рекомендуется, так как изменения будут потеряны после перезагрузки сети.
