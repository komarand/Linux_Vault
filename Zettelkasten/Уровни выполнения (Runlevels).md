# Уровни выполнения (Runlevels)

В классической системе SysVinit загрузка организовывалась через уровни выполнения (Runlevels) — состояния системы, определяющие набор запущенных служб. Они нумеровались от 0 до 6.

Основные уровни:
- 0: Остановка системы (Halt).
- 1 (или S): Однопользовательский режим (Recovery mode) без сети, для обслуживания.
- 3: Многопользовательский текстовый режим (сервер).
- 5: Многопользовательский графический режим (десктоп).
- 6: Перезагрузка (Reboot).

В `systemd` концепция runlevels заменена на цели (targets), например, уровень 3 — это `multi-user.target`, а уровень 5 — `graphical.target`.
