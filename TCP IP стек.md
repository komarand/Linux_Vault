# TCP/IP стек
#linux #networking #kernel #tcpip
Реализация стека протоколов TCP/IP в ядре Linux.
## Структура
- **L2 (канальный)**: обрабатывается драйверами сетевых карт, взаимодействие с [[MAC]]-адресами, [[ARP]].
- **L3 (сетевой)**: IP (IPv4/IPv6), маршрутизация, фрагментация.
- **L4 (транспортный)**: [[TCP]] (сегменты, состояния, окна), [[UDP]] (дейтаграммы).
- **Сокеты (Berkeley sockets)**: интерфейс между пользовательскими процессами и стеком.
## Буферы и очереди
- Для каждого интерфейса существуют очереди отправки (`qdisc`).
- Ядро использует буферы приёма/передачи (`rmem`, `wmem`), которые можно настраивать через `sysctl`.
## Просмотр статистики
```bash
cat /proc/net/dev       # статистика по интерфейсам
cat /proc/net/tcp       # активные TCP-соединения (raw)
ss -tulnp               # удобный просмотр сокетов
```
